**Workaround 2 — `is` operator cross-module (union-of-structs event dispatch):**

In `tamga_sdl3.orh`:
- Delete the `pub enum(u8) EventKind` enum entirely (lines 238-255)
- Delete the `pub struct Event` flat struct (lines 270-282) and all TAG constants (lines 217-232)
- Replace with a union type: `pub union Event { Quit: QuitEvent, KeyDown: KeyDownEvent, KeyUp: KeyUpEvent, MouseMotion: MouseMotionEvent, MouseButtonDown: MouseButtonEvent, MouseButtonUp: MouseButtonEvent, GamepadAxis: GamepadAxisEvent, GamepadButtonDown: GamepadButtonEvent, GamepadButtonUp: GamepadButtonEvent, GamepadAdded: GamepadConnectedEvent, GamepadRemoved: GamepadDisconnectedEvent, TextInput: TextInputEvent, WindowResized: WindowResizedEvent, WindowPixelResized: WindowPixelResizedEvent, WindowClose: WindowCloseEvent }`
- Rewrite `pollEvent()` to return `(null | Event)` using union construction syntax. Each tag branch constructs the union variant directly, e.g.: `return Event.KeyDown(KeyDownEvent(scancode: raw.getKeyScancode(), repeat: raw.getKeyRepeat(), timestamp: ts))`. This eliminates the massive per-branch boilerplate of zeroing all unused fields.
- Keep the TAG constants as internal (they still drive the raw->typed translation), OR use the RawEvent tag getTag() with if-chains as before but construct union variants instead of flat structs.

NOTE: The `is` operator cross-module fix means callers can now write `if(ev is tamga_sdl3.KeyDownEvent)` or `if(ev is KeyDownEvent)` with proper import. The exact `is` syntax for union variants in Orhon needs to match what the compiler expects. If Orhon unions use `ev is Event.KeyDown` pattern, use that. If it uses `ev is KeyDownEvent`, use that. The key point is the cross-module qualifier now works.
