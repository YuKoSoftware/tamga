  <action>
  Rewrite `src/TamgaSDL3/tamga_sdl3.zig` completely. The new file must contain ALL of the following sections. Per D-01, this is the internal bridge layer — no Orhon-facing types, only primitives and flat structs.

  **1. Imports and constants:**
  ```zig
  const c = @cImport(@cInclude("SDL3/SDL.h"));
  const std = @import("std");
  ```

  **2. RawEvent struct** — a flat discriminated struct that the Orhon layer reads via bridge. Per D-04/D-06, this replaces the old `Event` struct that exposed raw SDL_Event internals. Use tag constants (not an enum, since Orhon bridge reads these as u8):
  ```
  TAG_NONE = 0
  TAG_QUIT = 1
  TAG_KEY_DOWN = 2
  TAG_KEY_UP = 3
  TAG_MOUSE_MOTION = 4
  TAG_MOUSE_BUTTON_DOWN = 5
  TAG_MOUSE_BUTTON_UP = 6
  TAG_GAMEPAD_AXIS = 7
  TAG_GAMEPAD_BUTTON_DOWN = 8
  TAG_GAMEPAD_BUTTON_UP = 9
  TAG_GAMEPAD_ADDED = 10
  TAG_GAMEPAD_REMOVED = 11
  TAG_TEXT_INPUT = 12
  TAG_WINDOW_RESIZED = 13
  TAG_WINDOW_PIXEL_RESIZED = 14
  TAG_WINDOW_CLOSE = 15
  ```

  RawEvent fields:
  - `tag: u8` (discriminator)
  - `key_scancode: u32` (SDL scancode integer — Orhon layer maps to enum)
  - `key_repeat: bool`
  - `mouse_x: f32`, `mouse_y: f32` (absolute position)
  - `mouse_xrel: f32`, `mouse_yrel: f32` (relative delta)
  - `mouse_button: u8` (1=left, 2=middle, 3=right)
  - `mouse_down: bool`
  - `gamepad_which: u32` (joystick ID)
  - `gamepad_axis: u8` (axis index)
  - `gamepad_axis_value: i16` (-32768..32767)
  - `gamepad_button: u8` (button index)
  - `text: [32]u8` (UTF-8 null-terminated)
  - `window_w: i32`, `window_h: i32` (logical dimensions for RESIZED)
  - `pixel_w: i32`, `pixel_h: i32` (pixel dimensions for PIXEL_SIZE_CHANGED)
  - `timestamp: u64` (SDL event timestamp in nanoseconds)

  **3. `pollRawEvent(out: *RawEvent) bool` function:**
  - Call `c.SDL_PollEvent(&ev)`
  - Zero-initialize `out.*` (set tag=TAG_NONE, zero all fields)
  - Switch on `ev.type`:
    - `c.SDL_EVENT_QUIT` -> tag=TAG_QUIT
    - `c.SDL_EVENT_KEY_DOWN` -> tag=TAG_KEY_DOWN, fill key_scancode from `ev.key.scancode`, key_repeat from `ev.key.repeat`
    - `c.SDL_EVENT_KEY_UP` -> tag=TAG_KEY_UP, fill key_scancode, key_repeat
    - `c.SDL_EVENT_MOUSE_MOTION` -> tag=TAG_MOUSE_MOTION, fill mouse_x/y/xrel/yrel from `ev.motion`
    - `c.SDL_EVENT_MOUSE_BUTTON_DOWN` -> tag=TAG_MOUSE_BUTTON_DOWN, fill mouse_button from `ev.button.button`, mouse_down=true, mouse_x from `ev.button.x`, mouse_y from `ev.button.y`
    - `c.SDL_EVENT_MOUSE_BUTTON_UP` -> tag=TAG_MOUSE_BUTTON_UP, fill mouse_button, mouse_down=false, mouse_x/y
    - `c.SDL_EVENT_GAMEPAD_AXIS_MOTION` -> tag=TAG_GAMEPAD_AXIS, fill gamepad_which from `@intCast(ev.gaxis.which)`, gamepad_axis from `@intCast(ev.gaxis.axis)`, gamepad_axis_value from `ev.gaxis.value`
    - `c.SDL_EVENT_GAMEPAD_BUTTON_DOWN` -> tag=TAG_GAMEPAD_BUTTON_DOWN, fill gamepad_which from `@intCast(ev.gbutton.which)`, gamepad_button from `@intCast(ev.gbutton.button)`
    - `c.SDL_EVENT_GAMEPAD_BUTTON_UP` -> tag=TAG_GAMEPAD_BUTTON_UP, same fields
    - `c.SDL_EVENT_GAMEPAD_ADDED` -> tag=TAG_GAMEPAD_ADDED, fill gamepad_which from `@intCast(ev.gdevice.which)`
    - `c.SDL_EVENT_GAMEPAD_REMOVED` -> tag=TAG_GAMEPAD_REMOVED, same
    - `c.SDL_EVENT_TEXT_INPUT` -> tag=TAG_TEXT_INPUT, copy `std.mem.span(ev.text.text)` into `out.text[0..len]`, null-terminate
    - `c.SDL_EVENT_WINDOW_RESIZED` -> tag=TAG_WINDOW_RESIZED, fill window_w/h from `ev.window.data1`/`data2`
    - `c.SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED` -> tag=TAG_WINDOW_PIXEL_RESIZED, fill pixel_w/h from `ev.window.data1`/`data2`
    - `c.SDL_EVENT_WINDOW_CLOSE_REQUESTED` -> tag=TAG_WINDOW_CLOSE
    - else -> tag=TAG_NONE (ignore unknown events)
  - Set `out.timestamp = ev.common.timestamp` for all events (SDL3 provides this on every event)

  **4. Window struct** — keep the same pattern as current but add new methods:
  - `create(title, w, h, flags) anyerror!Window` — keep current implementation with null-terminated buffer copy. The `flags` parameter stays `u64` at the Zig level; the Orhon bitfield will map to this.
  - `destroy(self)` — same as current
  - `setTitle(self, title)` — same as current
  - `getWidth(self) i32` — same as current (logical size via `SDL_GetWindowSize`)
  - `getHeight(self) i32` — same as current
  - NEW: `getPixelWidth(self) i32` — use `c.SDL_GetWindowSizeInPixels` (per WIN-09, Pitfall 2)
  - NEW: `getPixelHeight(self) i32` — use `c.SDL_GetWindowSizeInPixels`
  - NEW: `getDisplayScale(self) f32` — use `c.SDL_GetWindowDisplayScale`
  - `getHandle(self) *anyopaque` — same as current (returns `@ptrCast(self.handle)`)
  - NEW: `setRelativeMouseMode(self, enabled: bool) bool` — call `c.SDL_SetWindowRelativeMouseMode(self.handle, enabled)` (per WIN-08)
  - NEW: `startTextInput(self) void` — call `c.SDL_StartTextInput(self.handle)` (per WIN-11, Pitfall 6)
  - NEW: `stopTextInput(self) void` — call `c.SDL_StopTextInput(self.handle)`

  **5. Cursor functions** (module-level, not Window methods — per SDL3 API):
  - `hideCursor() void` — call `c.SDL_HideCursor()`
  - `showCursor() void` — call `c.SDL_ShowCursor()`

  **6. Gamepad functions** (per WIN-06, Pitfall 4):
  - `openGamepad(id: u32) ?*anyopaque` — call `c.SDL_OpenGamepad(@intCast(id))`, return null on failure
  - `closeGamepad(handle: *anyopaque) void` — call `c.SDL_CloseGamepad(@ptrCast(@alignCast(handle)))`

  **7. Display info functions** (per WIN-14):
  - `getDisplayCount() i32` — use `c.SDL_GetDisplays(&count)`, free the array, return count
  - `getDisplayBounds(index: i32, out_x: *i32, out_y: *i32, out_w: *i32, out_h: *i32) bool` — use `c.SDL_GetDisplays` + `c.SDL_GetDisplayBounds` as shown in RESEARCH.md code example
  - `getDisplayContentScale(index: i32) f32` — use `c.SDL_GetDisplays` + `c.SDL_GetDisplayContentScale`
  - `getDisplayName(index: i32, out_buf: [*]u8, buf_len: usize) bool` — copy display name string into buffer

  **8. Lifecycle functions:**
  - `initPlatform() bool` — call `c.SDL_Init(c.SDL_INIT_VIDEO | c.SDL_INIT_EVENTS | c.SDL_INIT_GAMEPAD)`. Per Pitfall 4, always init GAMEPAD subsystem. Per D-02, user never sees init flags.
  - `quitPlatform() void` — call `c.SDL_Quit()`
  - `getErrorMessage() []const u8` — call `std.mem.span(c.SDL_GetError())`

  **9. Timing functions** (per LOOP-01/02, anti-pattern note):
  - `getTicksNS() u64` — call `c.SDL_GetTicksNS()` (nanoseconds, NOT milliseconds)
  - `delayNS(ns: u64) void` — call `c.SDL_DelayNS(ns)`

  **10. Remove old exports:**
  - Remove ALL old `pub const` flag/event/scancode constants (INIT_VIDEO, WINDOW_RESIZABLE, EVENT_QUIT, SCANCODE_ESCAPE, MOUSE_LEFT, etc.) — per D-02, these are internal to the Zig file now, used only in the switch statement
  - Remove old `Event` struct entirely — replaced by `RawEvent` + `pollRawEvent`
  - Remove old `Renderer` struct — the SDL2-style software renderer is not needed; Tamga uses Vulkan (VK3D/VK2D)
  - Remove old `init(flags)` and `quit()` — replaced by `initPlatform()` / `quitPlatform()`
  - Remove old `getTicks()` and `delay()` — replaced by nanosecond variants

  **Important:** Keep `const SdlError = error{SdlFailed};` for the error type. All functions that can fail should return `anyerror!T` using this error set.

  If any SDL3 C function has unexpected signature differences from what the research documents, log the discrepancy in `docs/bugs.txt` (per XC-05, D-14) and adapt.
